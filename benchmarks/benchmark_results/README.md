# Benchmark Results

Cross-dataset evaluation results produced by the benchmark pipeline.
See [benchmarks/README.md](../README.md) for runner documentation.

---

## Directory Classification

### Core data (do not delete)

These directories contain the primary metric CSVs and metadata that feed
statistical analysis and paper figure generation.

| Directory | Content | Size class |
|-----------|---------|------------|
| `crossdata/csv/` | Combined per-model/dataset/seed metric CSVs | ~338 MB |
| `crossdata/meta/` | Run metadata JSONs (hyperparams, timing) | Small |
| `external/csv/` + `external/meta/` | External baseline metrics | ~10 MB |
| `base/csv/` + `base/meta/` | Base ablation reference | ~2.5 MB |
| `sensitivity/csv/` | Hyperparameter sweep results | ~45 MB |
| `training/csv/` | Training dynamics metrics | ~47 MB |
| `preprocessing/csv/` | Preprocessing variant results | ~15 MB |
| `joint/csv/` | Joint-training results | ~27 MB |
| `interpretability/csv/` | Interpretability metrics | Small |
| `latent_dim/csv/` | Latent dimension sweep | Small |
| `scalability/csv/` | Scalability benchmarks | Small |
| `warmup/csv/` | Warmup ratio sweep | Small |
| `transfer/csv/` | Transfer learning results | Small |
| `statistical_exports/` | Wilcoxon, ranking, kNN, runtime CSVs + report | ~1.7 MB |

### Regenerable artefacts (safe to delete)

| Directory | Content | Size class | Regenerated by |
|-----------|---------|------------|----------------|
| `cache/` | Preprocessing `.h5ad` caches (hash-keyed) | **~7.6 GB** | Any benchmark runner (auto-created) |
| `crossdata/latents/` | Saved latent embeddings (`.npz`) | ~28 MB | `benchmark_crossdata.py` |
| `crossdata/plots/` | Per-run metric plots | ~6.4 MB | `benchmark_crossdata.py` |
| `models/` | Model checkpoints (used by bio-validation) | **~738 MB** | `benchmark_crossdata.py` / `benchmark_base.py` |

> **Tip:** To free ~8.3 GB of disk space, delete `cache/` and `models/`.
> They will be recreated on the next benchmark run.
> Only archive `models/` (not delete) if you need biological validation
> without re-training.

### Provenance / logs

| Directory | Content |
|-----------|---------|
| `_logs/run_manifest.jsonl` | Indexed record of all benchmark runs |
| `_logs/seed_pipeline.log` | Multi-seed pipeline execution log |

A symlink `run_manifest.jsonl` → `_logs/run_manifest.jsonl` exists at
the top level for backward compatibility with `benchmarks/run_manifest.py`.

### Legacy / exploratory

| Directory | Content |
|-----------|---------|
| `_legacy/extra_sample/` | Development-only extra-sample benchmark output |

## Data Flow

```
benchmark_*.py runners
    │
    ├──→ {subdir}/csv/       metric CSVs
    ├──→ {subdir}/meta/      run metadata
    ├──→ {subdir}/latents/   latent embeddings (optional)
    └──→ cache/              preprocessing caches
            │
            ▼
statistical_analysis.py
    │
    └──→ statistical_exports/   Wilcoxon, rankings, kNN, runtime CSVs
            │
            ▼
generate_paper_figures.py / figure_generators/
    │
    └──→ benchmarks/paper_figures/{dpmm,statistical}/
```

## Multiple CSV Generations

`crossdata/csv/` may contain multiple timestamped CSV files per dataset
(e.g., `results_blood_aged_20260214_*.csv`). The statistical analysis
and figure generators load the **most recent** file per dataset by
timestamp. Older generations are retained for reproducibility but can
be pruned if disk space is needed.
