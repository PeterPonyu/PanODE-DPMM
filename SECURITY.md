# Security Policy

## Supported versions

Security fixes are best-effort and are expected to land on the default branch first.

| Version | Supported |
| --- | --- |
| `main` | ✅ |
| older commits and ad hoc snapshots | ❌ |

## Reporting a vulnerability

Please do **not** open a public issue for a suspected vulnerability.

Instead:

1. use GitHub private vulnerability reporting if it is enabled for this repository, or
2. contact the maintainer through a private channel listed on the maintainer profile.

When reporting, include:

- a short description of the issue,
- affected files or commands,
- reproduction steps or proof of concept when safe to share, and
- any suggested mitigation.

You can expect an initial acknowledgement within a reasonable best-effort window, but this repository does not currently offer a formal response SLA.
